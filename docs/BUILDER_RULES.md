# BUILDER RULES

## Regras Gerais de Execução
O Builder deve SEMPRE seguir estas regras em qualquer tarefa.

---

## Regra 1 — Responder com Plano Antes de Executar
Antes de realizar qualquer alteração/código, SEMPRE responder com:
- plano curto
- objetivo da alteração
- arquivos que serão modificados

Nunca executar imediatamente sem plano.

---

## Regra 2 — Não Sair do Escopo
Executar SOMENTE o que foi pedido.

Nunca:
- adicionar features extras
- refatorar sem necessidade
- alterar outras áreas não solicitadas

---

## Regra 3 — Não Alterar Layout sem Permissão
Nunca alterar layout/visual existente sem autorização explícita.

Se uma tarefa impactar UI:
- explicar antes
- pedir confirmação

---

## Regra 4 — Priorizar Simplicidade
Sempre optar pela solução:
- mais simples
- mais limpa
- mais legível
- mais fácil de manter

Evitar overengineering.

---

## Regra 5 — Respeitar Fonte de Verdade
Sempre utilizar os arquivos .md dentro da pasta docs como fonte principal de contexto e verdade do projeto.

---

## Regra 6 — Mobile First Obrigatório
Toda interface deve ser pensada primeiro para mobile.

Desktop deve apenas adaptar.

---

## Regra 7 — Código Profissional
Todo código deve ser:
- organizado
- reutilizável
- modular
- escalável

---

## Regra 8 — Não Criar Complexidade Desnecessária
Evitar:
- abstrações prematuras
- lógica excessiva
- arquitetura exagerada

---

## Regra 9 — Confirmar Decisões Grandes
Se a tarefa impactar:
- arquitetura
- banco de dados
- UX principal
- estrutura global

O Builder deve parar e confirmar antes.
